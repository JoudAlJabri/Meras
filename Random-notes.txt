Bootstrap is a class-based CSS framework. It has no magic — it's literally just a huge CSS file full of pre-written styles. 
The only way to use those styles is by adding their specific class names to your HTML elements.
So instead of you writing:
css/* You would have to write all this yourself */
.my-button {
  display: flex;
  align-items: center;
  justify-content: center;
  padding: 12px 24px;
  border-radius: 8px;
  font-weight: 600;
  background-color: #3DB87A;
  color: white;
  border: none;
}
Bootstrap lets you just do:
jsx<button className="btn d-flex align-items-center justify-content-center px-4 py-3 rounded fw-semibold">
Each class name = one pre-written CSS rule. That's the whole system.

Breaking Down a Real Example
Let me take one div from the landing page and explain every single class:
jsx<div className="d-flex gap-3 flex-wrap mt-5">
ClassWhat it doesCSS equivalentd-flexmakes it a flexboxdisplay: flexgap-3space between childrengap: 1remflex-wrapwraps to next line on small screensflex-wrap: wrapmt-5margin topmargin-top: 3rem
And this one:
jsx<div className="col-lg-6 mb-5 mb-lg-0">
ClassWhat it doescol-lg-6takes up half the row on large screensmb-5margin bottom on all screensmb-lg-0removes that margin bottom on large screens

Why So Many Divs Specifically?
The divs are mostly for Bootstrap's grid system. Bootstrap thinks of every page as a 12-column grid. To use it you need this specific structure:
jsx{/* Level 1: container — centers everything and adds padding */}
<div className="container">

  {/* Level 2: row — creates the horizontal grid */}
  <div className="row">

    {/* Level 3: col — defines how wide each section is */}
    <div className="col-lg-6">
      {/* Left side content */}
    </div>

    <div className="col-lg-6">
      {/* Right side content */}
    </div>

  </div>
</div>
col-lg-6 means "take up 6 out of 12 columns on large screens" — so two of them sit side by side, each taking half the page. On mobile they automatically stack on top of each other.
Without these 3 levels (container → row → col) the grid simply doesn't work.

The Alternative (Why We Don't Do This)
You could write everything with custom CSS and fewer divs:
jsx// Cleaner JSX
<section className="hero">
  <div className="hero-left">
    <h1>Experience Your Major</h1>
  </div>
  <div className="hero-right">
    <div className="challenge-card">...</div>
  </div>
</section>
css/* But then you write ALL of this yourself */
.hero {
  display: flex;
  align-items: center;
  padding: 80px 0;
}
.hero-left {
  width: 50%;
  margin-bottom: 0;
}
.hero-right {
  width: 50%;
}
/* And then you handle mobile yourself: */
@media (max-width: 768px) {
  .hero {
    flex-direction: column;
  }
  .hero-left, .hero-right {
    width: 100%;
    margin-bottom: 2rem;
  }
}
```

Bootstrap handles all of that automatically with `col-lg-6`. The trade-off is more class names in your JSX but zero CSS to write yourself.

## Simple Mental Model

Think of Bootstrap classes as **shortcuts**:
```
"I want this div to be a flexbox 
 with centered items 
 and a gap between them
 and some padding
 and a white background
 and rounded corners
 and a shadow"

Without Bootstrap → write 7 CSS rules
With Bootstrap    → className="d-flex align-items-center gap-3 p-4 bg-white rounded shadow"
One line instead of a separate CSS file. That's the whole point.
